<guidelines>

**Фазы:**
- Целые числа для запланированной работы
- Десятичные (2.1, 2.2) для срочных вставок
- Каждая фаза имеет Goal, Depends on, Requirements, Success Criteria, Plans

**Success Criteria:**
- Наблюдаемое поведение с точки зрения пользователя
- Должно быть TRUE/FALSE (бинарная проверка)
- Не "сделать X", а "X работает так-то"

**Plans:**
- Нумерация: ФАЗА-ПЛАН (01-01, 01-02, 02-01...)
- Каждый план — атомарная единица работы
- Указывайте зависимости между планами

**Прогресс:**
- Таблица отслеживает completion каждой фазы
- Обновляется после выполнения каждого плана

</guidelines>
