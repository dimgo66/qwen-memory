# gsd:discuss-phase

Обсуждение деталей реализации фазы перед планированием.

## Что делает

1. Анализирует цель фазы из ROADMAP.md
2. Исследует технические подходы
3. Выявляет компромиссы и риски
4. Формулирует рекомендации
5. Обновляет ROADMAP.md с уточнениями

## Как использовать

```
gsd:discuss-phase 1
gsd:discuss-phase 2 --auto
gsd:discuss-phase 3 --analyze
```

## Флаги

| Флаг | Описание |
|------|----------|
| `--auto` | Автоматический режим: анализ → рекомендации без вопросов |
| `--analyze` | Глубокий анализ компромиссов с таблицей tradeoffs |

## Процесс

### Шаг 1: Контекст
- Читаю ROADMAP.md — цель фазы N, success criteria
- Читаю PROJECT.md — требования, ограничения
- Читаю STATE.md — текущее состояние

### Шаг 2: Исследование подходов

Для каждого аспекта фазы:
```
Подход А: [название]
- Плюсы: [...]
- Минусы: [...]
- Когда использовать: [...]
```

### Шаг 3: Компромиссы (с --analyze)

```markdown
## Tradeoff Analysis

| Критерий | Подход А | Подход Б | Подход В |
|----------|----------|----------|----------|
| Скорость | +++ | + | ++ |
| Сложность | Низкая | Средняя | Высокая |
| Масштаб | Хорошо | Отлично | Хорошо |
| Риск | Низкий | Средний | Высокий |

**Рекомендация:** Подход Б
**Почему:** Лучший баланс скорости и масштабируемости
```

### Шаг 4: Риски

```markdown
## Риски

1. **[Риск 1]** — [описание]
   - Вероятность: Низкая / Средняя / Высокая
   - Влияние: Низкое / Среднее / Высокое
   - Митигация: [как снизить]

2. **[Риск 2]** — [описание]
   - Вероятность: ...
```

### Шаг 5: Рекомендации

```markdown
## Рекомендации для Phase N

**Архитектура:**
- [Рекомендация 1]
- [Рекомендация 2]

**Технологии:**
- [Стек/библиотеки]

**Подходы:**
- [Паттерны, best practices]

**Избегать:**
- [Антипаттерны, рискованные решения]
```

### Шаг 6: Обновление ROADMAP.md

Добавляю секцию:
```markdown
### Phase N: Discussion Notes

**Обсуждено:** [дата]
**Режим:** --auto / --analyze / manual

**Решения:**
- [Решение 1]
- [Решение 2]

**Компромиссы:**
- [Tradeoff 1] → выбран [подход]

**Риски:**
- [Риск 1] — митигация: [...]
```

## Вывод

```
## gsd:discuss-phase N завершено

**Режим:** [manual / --auto / --analyze]

**Ключевые решения:**
1. [Решение 1]
2. [Решение 2]

**Рекомендованный подход:** [название]

**Следующий шаг:** gsd:plan-phase N
```

## Когда использовать

| Ситуация | Режим |
|----------|-------|
| Простая фаза, понятный стек | `gsd:discuss-phase N` (без флагов) |
| Фаза с рисками, нужен быстрый анализ | `--auto` |
| Сложная фаза, несколько подходов | `--analyze` |
| Уже обсуждали, переходим к плану | Пропустить, сразу `gsd:plan-phase N` |
