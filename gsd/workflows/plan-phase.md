# gsd:plan-phase

Детальное планирование фазы.

## Что делает

1. Читает ROADMAP.md для понимания цели фазы
2. Исследует технические детали (если нужно)
3. Создаёт детальные планы (plans) внутри фазы
4. Определяет зависимости между планами
5. Обновляет ROADMAP.md и STATE.md

## Как использовать

Напишите мне:
```
gsd:plan-phase 1
```
или
```
gsd:plan-phase 2
```

## Процесс

### Шаг 1: Контекст
- Читаю ROADMAP.md — цель фазы, success criteria
- Читаю STATE.md — текущее состояние
- Читаю PROJECT.md — требования

### Шаг 2: Research (если нужно)
- Исследую технологии, библиотеки, паттерны
- Проверяю совместимость
- Нахожу лучшие практики

### Шаг 3: Детальный план
Разбиваю фазу на планы:
```
План 01-01: [название]
- Задачи: [...]
- Зависимости: [...]
- Оценка: [...]

План 01-02: [название]
...
```

### Шаг 4: Верификация
- Проверяю полноту планов
- Проверяю зависимости
- Обновляю ROADMAP.md

### Шаг 5: Обновление состояния
- Обновляю STATE.md
- Готов к выполнению

## Результат

```markdown
### Phase 1: [Name]
**Goal**: [...]
**Plans**: 3 plans

Plans:
- [ ] 01-01: [детальное описание]
  - Dependencies: none
  - Estimated: 30 min
- [ ] 01-02: [детальное описание]
  - Dependencies: 01-01
  - Estimated: 45 min
- [ ] 01-03: [детальное описание]
  - Dependencies: 01-01, 01-02
  - Estimated: 60 min
```

## После завершения

Следующая команда: `gsd:execute-phase 1`
