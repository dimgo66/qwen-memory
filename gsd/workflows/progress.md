# gsd:progress

Показать текущее состояние проекта и ready планы.

## Что делает

- Читает `.planning/STATE.md`
- Показывает текущую фазу и план
- Отображает прогресс
- **Показывает dependency graph**
- **Определяет ready планы**
- Предлагает следующие шаги

## Как использовать

Напишите мне:
```
gsd:progress
```

## Что я покажу

```
## Текущая позиция

Фаза: [X] из [Y] ([Название])
План: [A] из [B]
Статус: [Ready / Planning / In Progress / Done]

Прогресс: [░░░░░░░░░░] X%

## Метрики

- Выполнено планов: N
- Среднее время: X мин
- Всего времени: X.X часов

## Dependency Graph (Active Phase)

```
01-01 ✓ → 01-02 🔄 → 01-04 ⏳
     ↘ 01-03 ⏳ ↗
```

## Ready to Execute ✓

Планы без блокировок:

| План | ID | Название | Зависимости |
|------|-----|----------|-------------|
| 01-03 | 01-03.d9e2 | Database | 01-01 ✓ |
| 01-05 | 01-05.e2f4 | Tests | 01-02 ✓ |

**Blocked:**
| 01-04 | 01-04.f1a3 | Frontend | 01-02 🔄, 01-03 ⏳ |

## Следующие шаги

1. Выполнить 01-03 (Database) — ready
2. Выполнить 01-05 (Tests) — ready
3. Ждать завершения 01-02 для 01-04
```

## Файлы

- `.planning/STATE.md` — текущее состояние
- `.planning/ROADMAP.md` — дорожная карта с dependency graph
- `.planning/PROJECT.md` — контекст проекта
