# gsd:ui-review

Ретроспективный визуальный аудит реализованного UI по шести направлениям.

## Что делает

1. Анализирует реализованный UI-код фазы N
2. Проводит аудит по 6 направлениям:
   - Визуальная иерархия
   - Консистентность
   - Доступность (a11y)
   - Адаптивность
   - Производительность
   - Взаимодействия
3. Создаёт отчёт с проблемами и рекомендациями
4. Предлагает план исправлений

## Как использовать

```
gsd:ui-review
gsd:ui-review 1
gsd:ui-review 2 --strict
```

## Шесть направлений аудита

### 1. Визуальная иерархия

**Проверяю:**
- [ ] Контраст размера между заголовками и текстом
- [ ] Использование whitespace для группировки
- [ ] Цветовые акценты для важных элементов
- [ ] Порядок чтения соответствует F-паттерну

**Оценка:** Pass / Partial / Fail

**Проблемы:**
```
✗ Заголовки H1/H2 недостаточно отличаются
✗ Кнопки CTA теряются на фоне других элементов
```

### 2. Консистентность

**Проверяю:**
- [ ] Единые отступы (spacing tokens)
- [ ] Согласованные цвета (color tokens)
- [ ] Одинаковые стили кнопок
- [ ] Единая типографика

**Оценка:** Pass / Partial / Fail

**Проблемы:**
```
✗ 5 разных значений padding для кнопок
✗ Цвет hover отличается на similar элементах
```

### 3. Доступность (a11y)

**Проверяю:**
- [ ] Контраст цветов ≥ 4.5:1 (WCAG AA)
- [ ] Focus indicators видны
- [ ] ARIA labels присутствуют
- [ ] Keyboard navigation работает
- [ ] Alt text у изображений

**Оценка:** Pass / Partial / Fail

**Проблемы:**
```
✗ Контраст текста 3.2:1 (требуется 4.5:1)
✗ Отсутствует focus outline на кнопках
✗ Нет aria-label у иконок-кнопок
```

### 4. Адаптивность

**Проверяю:**
- [ ] Mobile (<640px) работает
- [ ] Tablet (640-1024px) оптимизирован
- [ ] Desktop (>1024px) использует пространство
- [ ] Touch targets ≥ 44px

**Оценка:** Pass / Partial / Fail

**Проблемы:**
```
✗ На mobile текст выходит за границы
✗ Hamburger menu не открывается на tablet
```

### 5. Производительность

**Проверяю:**
- [ ] Изображения оптимизированы (WebP, lazy loading)
- [ ] CSS/JS минифицированы
- [ ] Критический CSS инлайнится
- [ ] Шрифты загружаются эффективно

**Оценка:** Pass / Partial / Fail

**Проблемы:**
```
✗ Изображения 2MB без сжатия
✗ Блокирующий JS 500KB в head
```

### 6. Взаимодействия

**Проверяю:**
- [ ] Hover состояния видны
- [ ] Active состояния реагируют
- [ ] Loading состояния показаны
- [ ] Error состояния понятны
- [ ] Анимации плавные (60fps)

**Оценка:** Pass / Partial / Fail

**Проблемы:**
```
✗ Нет loading state для форм
✗ Анимация дёргается на mobile
```

## Процесс

### Шаг 1: Анализ кода
- Читаю UI-компоненты
- Проверяю CSS/стили
- Тестирую взаимодействия

### Шаг 2: Аудит по 6 направлениям

Заполняю таблицу:

```markdown
## Audit Summary

| Направление | Оценка | Проблем |
|-------------|--------|---------|
| Визуальная иерархия | Partial | 2 |
| Консистентность | Pass | 0 |
| Доступность | Fail | 3 |
| Адаптивность | Partial | 2 |
| Производительность | Pass | 1 |
| Взаимодействия | Partial | 2 |

**Общая оценка:** 2/6 Pass, 3/6 Partial, 1/6 Fail
```

### Шаг 3: Детальный отчёт

```markdown
## Детальные проблемы

### [Направление]

**Проблема 1:** [Описание]
- **Где:** `path/to/file.vue:42`
- **Серьёзность:** High / Medium / Low
- **WCAG:** [если применимо]
- **Как исправить:** [рекомендация]

**Проблема 2:** ...
```

### Шаг 4: План исправлений

```markdown
## План исправлений

**Priority 1 (High):**
- [ ] Исправить контраст текста
- [ ] Добавить focus indicators

**Priority 2 (Medium):**
- [ ] Унифицировать отступы
- [ ] Исправить mobile layout

**Priority 3 (Low):**
- [ ] Оптимизировать анимации
```

### Шаг 5: Вывод

```
## gsd:ui-review завершено

**Общая оценка:** X/6 Pass, Y/6 Partial, Z/6 Fail

**Критические проблемы:** N
**Рекомендации:** M

**Следующий шаг:**
- gsd:do: исправить Priority 1 проблемы
- gsd:verify-work — после исправлений
```

## Флаги

| Флаг | Описание |
|------|----------|
| `--strict` | Строгий режим (WCAG AAA вместо AA) |
| `--report` | Экспорт полного отчёта в `.planning/audits/` |
| `--ci` | Формат для CI/CD (JSON output) |

## Вывод отчёта

```bash
# С флагом --report
.planning/
└── audits/
    └── ui-review-phase-N-YYYY-MM-DD.md
```

## Когда использовать

| Ситуация | Команда |
|----------|---------|
| После UI-фазы | `gsd:ui-review N` |
| Перед релизом | `gsd:ui-review --report` |
| Accessibility критичен | `gsd:ui-review --strict` |
| В CI/CD pipeline | `gsd:ui-review --ci` |
