# gsd:unified-workflow

9-шаговый дисциплинированный процесс разработки.

## Что делает

Объединяет все GSD команды в единый workflow от задачи до завершения:

1. **TASK** — Создание задачи
2. **BRAINSTORM** — Исследование подходов
3. **PLAN** — Детальное планирование
4. **ISOLATE** — Изоляция ветки
5. **IMPLEMENT** — TDD реализация
6. **REVIEW** — Code review
7. **VERIFY** — Верификация
8. **FINISH** — Завершение ветки
9. **CLOSE** — Закрытие задачи

## Как использовать

Напишите мне:
```
gsd:unified-workflow
```
или для конкретной фазы/задачи:
```
gsd:unified-workflow 1
```

## 9-Шаговый Процесс

### Шаг 1: TASK — Создание задачи

**Цель:** Чётко определить задачу перед началом работы.

**Команды:**
```
gsd:requirements-spec N     # Уточнение требований
gsd:discuss-phase N         # Обсуждение
```

**Чек-лист:**
- [ ] Задача имеет ясную цель
- [ ] Требования задокументированы
- [ ] Acceptance criteria определены
- [ ] Тест-кейсы подготовлены

**Результат:** REQUIREMENTS.md с acceptance criteria

---

### Шаг 2: BRAINSTORM — Исследование подходов

**Цель:** Исследовать альтернативные решения.

**Команды:**
```
gsd:discuss-phase N --analyze  # Анализ компромиссов
```

**Чек-лист:**
- [ ] Рассмотрено ≥ 2 подхода
- [ ] Tradeoff таблица заполнена
- [ ] Выбран оптимальный вариант
- [ ] Риски идентифицированы

**Результат:** Документированные компромиссы, выбранное решение

---

### Шаг 3: PLAN — Детальное планирование

**Цель:** Разбить задачу на выполнимые планы.

**Команды:**
```
gsd:plan-phase N          # Планирование с графом зависимостей
gsd:writing-plans N       # Структурированные планы
gsd:dependency-graph N    # Построить граф
```

**Чек-лист:**
- [ ] Планы имеют hash-based ID (01-01.a3f8)
- [ ] Зависимости определены
- [ ] Критический путь найден
- [ ] Ready планы идентифицированы

**Результат:** ROADMAP.md с dependency graph

---

### Шаг 4: ISOLATE — Изоляция ветки

**Цель:** Создать изолированное окружение для разработки.

**Команды:**
```bash
git worktree add ../feature-branch feature-branch
# или
git checkout -b feature/phase-N
```

**Чек-лист:**
- [ ] Ветка создана от актуального main
- [ ] Имя ветки отражает задачу
- [ ] Окружение настроено

**Результат:** Isolated feature branch

---

### Шаг 5: IMPLEMENT — TDD реализация

**Цель:** Реализация с обязательным TDD.

**Команды:**
```
gsd:execute-phase N       # Выполнение с TDD
gsd:ready                 # Проверка готовых планов
```

**TDD Цикл (RED-GREEN-REFACTOR):**

1. **RED** — Написать failing тест
   - [ ] Тест покрывает happy path
   - [ ] Тест покрывает edge cases
   - [ ] Тест покрывает error cases
   - [ ] Первый запуск: FAIL ✓

2. **GREEN** — Минимальная реализация
   - [ ] Код написан для прохождения тестов
   - [ ] Все тесты PASS

3. **REFACTOR** — Улучшение кода
   - [ ] Код рефакторен без изменения поведения
   - [ ] Тесты всё ещё PASS
   - [ ] Коммит сделан

**Правила:**
- ❌ **Не писать код без failing-теста**
- ❌ **Не коммитить без зелёных тестов**

**Результат:** Реализованные планы, все тесты PASS

---

### Шаг 6: REVIEW — Code review

**Цель:** Обнаружить проблемы до merge.

**Команды:**
```
skill:review              # Автоматическое ревью
gsd:ship --dry-run        # Пре-проверка
```

**Чек-лист ревью:**
- [ ] Correctness — логика верна
- [ ] Security — нет уязвимостей
- [ ] Code quality — чистый код
- [ ] Performance — нет проблем
- [ ] Тесты покрывают критичные участки
- [ ] Нет хардкодов, секретов
- [ ] Обработка ошибок корректна

**Правила:**
- ❌ **Не merge без code review**

**Результат:** Approved code, список исправлений (если есть)

---

### Шаг 7: VERIFY — Верификация

**Цель:** Подтвердить соответствие требованиям.

**Команды:**
```
gsd:verify-work N         # UAT + Verification
```

**Verification Checklist:**
- [ ] Все acceptance criteria проверены
- [ ] Все тесты PASS (100%)
- [ ] Покрытие ≥ 80%
- [ ] Lint проходит без ошибок
- [ ] TypeScript компилируется
- [ ] Нет console.error/warn в логах
- [ ] Производительность в норме
- [ ] Безопасность проверена

**Правила:**
- ❌ **Не声称 "готово" без верификации**

**Результат:** UAT отчёт, все критерии PASS

---

### Шаг 8: FINISH — Завершение ветки

**Цель:** Merge изменений в main.

**Команды:**
```bash
git checkout main
git merge feature/phase-N
git push origin main
# или
git worktree remove ../feature-branch
```

**Чек-лист:**
- [ ] Все тесты на CI прошли
- [ ] Конфликты разрешены
- [ ] Коммиты squash'ены (если нужно)
- [ ] Ветка удалена

**Результат:** Changes merged to main

---

### Шаг 9: CLOSE — Закрытие задачи

**Цель:** Завершить задачу, обновить статусы.

**Команды:**
```
gsd:ship N                # Публикация
gsd:compact N             # Архивация (опционально)
gsd:next                  # Переход к следующей
```

**Чек-лист:**
- [ ] Фаза отмечена shipped
- [ ] STATE.md обновлён
- [ ] ROADMAP.md обновлён
- [ ] Архив создан (если compact)

**Результат:** Task closed, ready for next

---

## Полный Workflow

```
┌─────────────────────────────────────────────────────────────┐
│                    UNIFIED WORKFLOW                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. TASK         → gsd:requirements-spec N                  │
│       │                                                        │
│  2. BRAINSTORM   → gsd:discuss-phase N --analyze            │
│       │                                                        │
│  3. PLAN         → gsd:plan-phase N                         │
│       │                gsd:dependency-graph N                │
│       │                                                        │
│  4. ISOLATE      → git worktree / checkout -b               │
│       │                                                        │
│  5. IMPLEMENT    → gsd:execute-phase N (TDD)                │
│       │                RED → GREEN → REFACTOR → COMMIT       │
│       │                                                        │
│  6. REVIEW       → skill:review                             │
│       │                                                        │
│  7. VERIFY       → gsd:verify-work N                        │
│       │                                                        │
│  8. FINISH       → git merge / push                         │
│       │                                                        │
│  9. CLOSE        → gsd:ship N                               │
│                       gsd:next                               │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## Золотые Правила

| Правило | Описание |
|---------|----------|
| ❌ **Нет коду без failing-теста** | TDD обязательно |
| ❌ **Нет мерджу без review** | Code review обязательно |
| ❌ **Нет "готово" без верификации** | Verification checklist |
| ❌ **Нет работы без задачи** | Требования задокументированы |
| ✅ **Evidence before claims** | Доказательства вместо утверждений |
