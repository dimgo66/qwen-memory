# gsd:dependency-graph

Управление графом зависимостей между планами.

## Что делает

1. Визуализирует зависимости между планами фазы
2. Определяет критический путь
3. Находит блокировки и узкие места
4. Показывает готовые к выполнению планы (ready)

## Как использовать

Напишите мне:
```
gsd:dependency-graph 1
```
или
```
gsd:ready
```

## Процесс

### Шаг 1: Чтение зависимостей

**Анализирую ROADMAP.md:**
- Все планы фазы
- Зависимости каждого плана
- Статус выполнения

### Шаг 2: Построение графа

**Формат графа:**

```
## Dependency Graph: Phase 1

### Визуализация

01-01 (Auth)
    ├── 01-02 (API)
    │       ├── 01-04 (Frontend)
    │       └── 01-05 (Tests)
    └── 01-03 (Database)
            └── 01-04 (Frontend)

### Критический путь
01-01 → 01-02 → 01-04 → 01-06
Длина: 4 плана, ~180 мин
```

### Шаг 3: Ready Check

**Готовые планы (без блокировок):**

```
## Ready to Execute ✓

Планы без незавершённых зависимостей:

| План | Название | Зависит от | Статус зависимостей |
|------|----------|------------|---------------------|
| 01-01 | Auth Setup | none | ✓ Ready |
| 01-03 | Database | 01-01 | ✓ Done |

**Blocked:**
| План | Название | Блокируется |
|------|----------|-------------|
| 01-04 | Frontend | 01-02, 01-03 |
```

### Шаг 4: Анализ узких мест

**Bottleneck Analysis:**

```
## Bottlenecks

⚠️ 01-02 (API) — блокирует 3 плана
   - 01-04 (Frontend)
   - 01-05 (Tests)
   - 01-06 (Integration)

Рекомендация: Выполнить в первую очередь
```

## Формат хранения

**В ROADMAP.md:**

```markdown
### Phase 1: [Name]

**Dependency Graph:**
```
01-01 → 01-02 → 01-04
    ↘ 01-03 ↗
```

**Dependencies Table:**

| Plan | ID | Depends On | Blocks | Status |
|------|-----|------------|--------|--------|
| Auth Setup | 01-01.a3f8 | none | 01-02, 01-03 | done |
| API | 01-02.b7c1 | 01-01 | 01-04, 01-05 | in_progress |
| Database | 01-03.d9e2 | 01-01 | 01-04 | pending |
| Frontend | 01-04.f1a3 | 01-02, 01-03 | 01-06 | pending |
```
```

## Hash-based ID

**Генерация ID для планов:**

```
План 01-01 → 01-01.a3f8
План 01-02 → 01-02.b7c1
```

**Преимущества:**
- Уникальная идентификация
- Нет конфликтов при merge
- Легко ссылаться в коммитах

## Команды

| Команда | Описание |
|---------|----------|
| `gsd:dependency-graph 1` | Построить граф зависимостей |
| `gsd:ready` | Показать готовые планы |
| `gsd:critical-path 1` | Показать критический путь |
| `gsd:bottlenecks 1` | Найти узкие места |

## После выполнения

**Используйте для:**
- Выбора следующего плана (`gsd:execute-phase`)
- Оптимизации порядка выполнения
- Понимания блокировок
