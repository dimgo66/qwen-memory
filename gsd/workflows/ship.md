# gsd:ship

Завершение работы и публикация.

## Что делает

1. Финализирует выполненную фазу
2. Создаёт коммит/PR/релиз
3. Обновляет STATE.md
4. Готовится к следующей фазе

## Как использовать

Напишите мне:
```
gsd:ship
```
или для конкретной фазы:
```
gsd:ship 1
```

## Процесс

### Шаг 1: Проверка готовности
- Все планы фазы выполнены
- UAT пройден
- Нет открытых issues

### Шаг 2: Финализация

**Git:**
```bash
git add .
git commit -m "Ship Phase 1: [название]"
git push
```

**PR (если нужно):**
- Создаю PR с описанием
- Добавляю reviewers

**Release (если нужно):**
- Создаю тег
- Публикую релиз

### Шаг 3: Обновление состояния

**ROADMAP.md:**
```
- [x] **Phase 1: [Name]** — shipped
```

**STATE.md:**
```
Phase: 2 из 4 (Ready to plan)
Progress: [██░░░░░░░░] 25%
```

**PROJECT.md:**
```
### Validated
- ✓ [Requirement] — Phase 1
```

### Шаг 4: Следующие шаги

```
## Готово!

Phase 1: [название] — shipped ✓

Следующие шаги:
1. gsd:plan-phase 2 — планирование следующей фазы
2. gsd:progress — посмотреть общее состояние
```

## Опции

**С коммитом:**
```
gsd:ship --commit
```

**С PR:**
```
gsd:ship --pr
```

**С релизом:**
```
gsd:ship --release
```
