# gsd:ship

Завершение работы и публикация с code review.

## Что делает

1. Финализирует выполненную фазу
2. **Проводит code review** перед публикацией
3. Создаёт коммит/PR/релиз
4. Обновляет STATE.md
5. Готовится к следующей фазе

## Как использовать

Напишите мне:
```
gsd:ship
```
или для конкретной фазы:
```
gsd:ship 1
```

## Процесс

### Шаг 1: Проверка готовности
- Все планы фазы выполнены
- UAT пройден
- **Все тесты проходят (PASS)**
- Нет открытых issues

### Шаг 1.5: Code Review (обязательно)

**Автоматическое ревью:**
- Запускаю `skill:review` для изменённых файлов
- Проверяю:
  - ✅ Correctness — логика верна
  - ✅ Security — нет уязвимостей
  - ✅ Code quality — чистый код
  - ✅ Performance — нет проблем с производительностью

**Чек-лист ревью:**
- [ ] Код следует лучшим практикам проекта
- [ ] Нет дублирования (DRY)
- [ ] Имена понятные
- [ ] Сложность приемлемая
- [ ] Тесты покрывают критичные участки
- [ ] Нет хардкодов, секретов
- [ ] Обработка ошибок корректна

**Если найдены проблемы:**
- Создаю список замечаний
- Исправляю критичные issues
- Повторяю ревью

### Шаг 2: Финализация

**Git:**
```bash
git add .
git commit -m "Ship Phase 1: [название]"
git push
```

**PR (если нужно):**
- Создаю PR с описанием
- Добавляю reviewers

**Release (если нужно):**
- Создаю тег
- Публикую релиз

### Шаг 3: Обновление состояния

**ROADMAP.md:**
```
- [x] **Phase 1: [Name]** — shipped
```

**STATE.md:**
```
Phase: 2 из 4 (Ready to plan)
Progress: [██░░░░░░░░] 25%
```

**PROJECT.md:**
```
### Validated
- ✓ [Requirement] — Phase 1
```

### Шаг 4: Следующие шаги

```
## Готово!

Phase 1: [название] — shipped ✓

Следующие шаги:
1. gsd:plan-phase 2 — планирование следующей фазы
2. gsd:progress — посмотреть общее состояние
```

## Опции

**С коммитом:**
```
gsd:ship --commit
```

**С PR:**
```
gsd:ship --pr
```

**С релизом:**
```
gsd:ship --release
```
