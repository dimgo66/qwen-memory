# gsd:new-project

Инициализация нового проекта через Qwen Code.

## Что делает

1. Задаёт уточняющие вопросы о проекте
2. (Опционально) Исследует домен
3. Создаёт требования
4. Формирует roadmap по фазам

## Создаваемые файлы

```
.planning/
├── PROJECT.md          — Контекст проекта
├── REQUIREMENTS.md     — Требования
├── ROADMAP.md          — Дорожная карта
├── STATE.md            — Состояние
└── config.json         — Конфигурация
```

## Как использовать

Напишите мне:
```
gsd:new-project
```

Или с идеей:
```
gsd:new-project: [ваша идея]
```

## Процесс

### Шаг 1: Вопросы
Я задам вопросы для понимания:
- Что строим?
- Для кого?
- Какая основная ценность?
- Какие есть ограничения?

### Шаг 2: Research (опционально)
Если нужно — исследую домен, конкурентов, лучшие практики.

### Шаг 3: Требования
Сформулирую требования в формате:
- Validated (уже проверенные)
- Active (текущие)
- Out of Scope (вне скоупа)

### Шаг 4: Roadmap
Создам план по фазам с:
- Целями каждой фазы
- Критериями успеха
- Планами внутри фаз

## После завершения

Следующая команда: `gsd:plan-phase 1`
