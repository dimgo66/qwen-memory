# gsd:writing-plans

Структурированное создание планов с компромиссами.

## Что делает

1. Анализирует требования фазы
2. **Исследует альтернативные подходы**
3. **Создаёт tradeoff таблицу**
4. Выбирает оптимальное решение
5. Создаёт детальные планы

## Как использовать

Напишите мне:
```
gsd:writing-plans 1
```
или
```
gsd:discuss-phase 1 --analyze
```

## Процесс

### Шаг 1: Анализ требований

**Читаю:**
- REQUIREMENTS.md — acceptance criteria
- PROJECT.md — ограничения, контекст
- ROADMAP.md — цель фазы

### Шаг 2: Исследование подходов

**Для каждой сложной задачи исследую:**

| Подход | Плюсы | Минусы | Когда использовать |
|--------|-------|--------|-------------------|
| **Вариант A** | Быстро, просто | Масштабируемость | Прототип, MVP |
| **Вариант B** | Надёжно, масштабируемо | Сложнее, дольше | Production |
| **Вариант C** | Гибко, кастомизируемо | Поддержка | Уникальные требования |

### Шаг 3: Tradeoff Таблица

**Пример анализа:**

```
## Выбор архитектуры для Phase 1

| Критерий | Вариант A | Вариант B | Вариант C |
|----------|-----------|-----------|-----------|
| Скорость | ⭐⭐⭐ | ⭐⭐ | ⭐ |
| Надёжность | ⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| Сложность | ⭐⭐⭐ | ⭐⭐ | ⭐ |
| Масштабирование | ⭐ | ⭐⭐⭐ | ⭐⭐ |
| Поддержка | ⭐⭐ | ⭐⭐⭐ | ⭐ |

**Рекомендация:** Вариант B
**Причина:** Production-ready, долгосрочная поддержка
```

### Шаг 4: Детальные планы

**Создаю планы с зависимостями:**

```markdown
### План 01-01: [Название]

**Цель**: [что делаем]

**Подход**: [выбранный вариант]

**Задачи**:
1. [задача 1]
2. [задача 2]
3. [задача 3]

**Зависимости**:
- Блокирует: 01-02, 01-03
- Зависит от: none

**Тесты**:
- TC-01-01: [happy path]
- TC-01-02: [edge case]
- TC-01-03: [error case]

**Оценка**: 45 мин

**Риски**:
- [риск 1] — митигация: [как избежать]
- [риск 2] — митигация: [как избежать]
```

### Шаг 5: Верификация планов

**Чек-лист:**
- [ ] Все планы имеют чёткую цель
- [ ] Зависимости определены
- [ ] Тест-кейсы указаны
- [ ] Оценка времени реалистична
- [ ] Риски идентифицированы
- [ ] Компромиссы задокументированы

## Результат

```markdown
## Plans Complete ✓

**Phase 1**: [Название]
**Планов**: 5

**Архитектурные решения**:
- Выбор: [вариант]
- Причина: [обоснование]
- Альтернативы: [отклонённые варианты]

**Зависимости**:
```
01-01 → 01-02 → 01-04
    ↘ 01-03 ↗
```

Готово к выполнению: `gsd:execute-phase 1`
```

## После завершения

Следующая команда: `gsd:execute-phase 1`

## Связь с другими воркфлоу

- **До**: `gsd:requirements-spec N`
- **После**: `gsd:execute-phase N`
- **Альтернатива**: `gsd:plan-phase N` (без глубокого анализа)
