# gsd:execute-phase

Выполнение планов фазы.

## Что делает

1. Читает ROADMAP.md с планами фазы
2. Выполняет планы параллельными волнами
3. Обновляет STATE.md после каждого плана
4. Отмечает выполненные планы в ROADMAP.md

## Как использовать

Напишите мне:
```
gsd:execute-phase 1
```

## Процесс

### Шаг 1: Подготовка
- Читаю ROADMAP.md — планы фазы
- Читаю STATE.md — текущий прогресс
- Определяю следующий план для выполнения

### Шаг 2: Выполнение волнами

**Волна 1** (параллельно, без зависимостей):
- План 01-01
- План 01-02

**Волна 2** (зависит от Волны 1):
- План 01-03

**Волна 3** (зависит от Волны 2):
- План 01-04

### Шаг 3: После каждого плана
- Обновляю STATE.md (прогресс, метрики)
- Отмечаю план в ROADMAP.md
- Коммичу изменения (если git)

### Шаг 4: Завершение фазы
- Проверяю все критерии успеха
- Обновляю STATE.md
- Готов к verify-work

## Формат отчёта

```
## Выполнение Phase 1

### Волна 1
✓ План 01-01: [название] — 25 мин
✓ План 01-02: [название] — 30 мин

### Волна 2
✓ План 01-03: [название] — 45 мин

### Итого
- Выполнено: 3/3 плана
- Время: 100 мин
- Статус: Phase complete
```

## После завершения

Следующая команда: `gsd:verify-work`
