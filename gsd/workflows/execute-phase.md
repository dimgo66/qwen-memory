# gsd:execute-phase

Выполнение планов фазы с TDD.

## Что делает

1. Читает ROADMAP.md с планами фазы
2. **Сначала пишет тесты** (TDD — Red Green Refactor)
3. Выполняет планы параллельными волнами
4. Обновляет STATE.md после каждого плана
5. Отмечает выполненные планы в ROADMAP.md

## Как использовать

Напишите мне:
```
gsd:execute-phase 1
```

## Процесс

### Шаг 1: Подготовка
- Читаю ROADMAP.md — планы фазы
- Читаю STATE.md — текущий прогресс
- Определяю следующий план для выполнения
- **Проверяю тестовый фреймворк проекта**

### Шаг 2: TDD для каждого плана

**Для каждого плана выполняю цикл RED-GREEN-REFACTOR:**

#### RED — Сначала тесты
1. Пишу **комплексные тесты** до реализации:
   - Happy path сценарии
   - Edge cases (граничные значения, пустые данные)
   - Error cases (обработка ошибок)
2. Запускаю тесты — убеждаюсь что **FAIL** (красные)
3. Сохраняю тесты в репозиторий

#### GREEN — Минимальная реализация
1. Пишу **минимальный код** для прохождения тестов
2. Запускаю тесты снова
3. Если FAIL → анализирую ошибку → вношу изменения → повторяю тест
4. Повторяю пока все тесты не пройдут (PASS)

#### REFACTOR — Улучшение кода
1. Рефакторю код (сохраняя тесты зелёными)
2. Запускаю тесты финально — убеждаюсь что PASS
3. Коммичу изменения

### Шаг 3: После каждого плана
- Обновляю STATE.md (прогресс, метрики)
- Отмечаю план в ROADMAP.md
- **Прикрепляю результаты тестов**
- Коммичу изменения (если git)

### Шаг 4: Завершение фазы
- Проверяю все критерии успеха
- **Запускаю полный тест-сьют**
- Обновляю STATE.md
- Готов к verify-work

## Формат отчёта

```

### Волна 1
✓ План 01-01: [название]
  - Tests: 12 passed (RED → GREEN → REFACTOR)
  - Время: 25 мин

✓ План 01-02: [название]
  - Tests: 8 passed
  - Время: 30 мин

### Волна 2
✓ План 01-03: [название]
  - Tests: 15 passed
  - Время: 45 мин

### Итого
- Выполнено: 3/3 плана
- Тестов: 35 passed ✓
- Время: 100 мин
- Статус: Phase complete
```

## TDD Checklist

Перед завершением плана:
- [ ] Тесты написаны ДО кода
- [ ] Тесты покрывают happy path
- [ ] Тесты покрывают edge cases
- [ ] Тесты покрывают error cases
- [ ] Первый запуск: FAIL ✓
- [ ] Код написан для прохождения тестов
- [ ] Финальный запуск: PASS ✓
- [ ] Рефакторинг выполнен
- [ ] Тесты закоммичены вместе с кодом

## После завершения

Следующая команда: `gsd:verify-work`
