# GSD Commands для Qwen Code

Полный список команд адаптированной версии Get Shit Done v1.29.0

## Автоматическое подключение

**MCP серверы** и **Skills** подключаются автоматически в зависимости от воркфлоу:

- **MCP:** filesystem, git, memory, postgres, puppeteer, playwright
- **Skills:** anthropic-frontend-design, web-design-guidelines, vercel-react-best-practices, review

Пользователю не нужно указывать их вручную — я сам подключу нужные!

## Core Workflow

| Команда | Описание |
|---------|----------|
| `gsd:new-project` | Инициализация: вопросы → research → требования → roadmap |
| `gsd:map-codebase` | Анализ существующего проекта перед инициализацией |
| `gsd:discuss-phase [N]` | Обсуждение деталей реализации фазы N |
| `gsd:discuss-phase [N] --auto` | Автоматический анализ без вопросов |
| `gsd:discuss-phase [N] --analyze` | Глубокий анализ компромиссов (tradeoffs) |
| `gsd:plan-phase [N]` | Research + план + верификация для фазы N |
| `gsd:execute-phase [N]` | Выполнение всех планов фазы N |
| `gsd:verify-work [N]` | Ручное тестирование (UAT) |
| `gsd:ship [N]` | Создание релиза/PR из проверенной работы |
| `gsd:next` | Авто-переход к следующему шагу |

## Дизайн пользовательского интерфейса

| Команда | Описание |
|---------|----------|
| `gsd:ui-phase [N]` | Сгенерируйте контракт на разработку UI (UI-SPEC.md) |
| `gsd:ui-phase [N] --detailed` | Расширенная спецификация с микро-взаимодействиями |
| `gsd:ui-review [N]` | Ретроспективный визуальный аудит по 6 направлениям |
| `gsd:ui-review [N] --strict` | Строгий режим (WCAG AAA) |
| `gsd:ui-review [N] --report` | Экспорт полного отчёта в `.planning/audits/` |

## Навигация и управление

| Команда | Описание |
|---------|----------|
| `gsd:progress` | Где я? Что дальше? |
| `gsd:next` | Автоматическое определение состояния и переход к следующему шагу |
| `gsd:manager` | Интерактивный командный центр для управления несколькими фазами |
| `gsd:manager --overview` | Dashboard проекта без меню |
| `gsd:manager --interactive` | Полноценный интерактивный режим |
| `gsd:help` | Показать все команды и руководство по использованию |
| `gsd:update` | Обновить GSD, ознакомиться с изменениями |
| `gsd:settings` | Конфигурация профиля модели и агентов |
| `gsd:set-profile <profile>` | Переключение профиля (quality/balanced/budget/inherit) |
| `gsd:stats` | Статистика проекта — фазы, планы, требования, git метрики |
| `gsd:health` | Валидация .planning/ директории |
| `gsd:health --repair` | Автоматическое исправление проблем |

## Быстрые задачи

| Команда | Описание |
|---------|----------|
| `gsd:quick [задача]` | Быстрое выполнение разовых задач |
| `gsd:quick [задача] --discuss` | С обсуждением перед планированием |
| `gsd:quick [задача] --research` | С исследованием подходов |
| `gsd:quick [задача] --full` | С plan-checking + верификацией |
| `gsd:do [задача]` | Выполнить конкретную задачу |
| `gsd:note [заметка]` | Захватить идею в pending todos |

## Управление фазами

| Команда | Описание |
|---------|----------|
| `gsd:add-phase [N] [name]` | Добавить новую фазу в roadmap |
| `gsd:remove-phase [N]` | Удалить фазу из roadmap |
| `gsd:insert-phase [N] [name]` | Вставить срочную фазу (decimal, e.g. 2.1) |
| `gsd:pause-work` | Временно приостановить работу |
| `gsd:resume-work` | Возобновить работу после паузы |

## Исследования и анализ

| Команда | Описание |
|---------|----------|
| `gsd:map-codebase` | Анализ существующей кодовой базы перед new-project |

## Пример сессии

```
Вы: gsd:new-project
Qwen: Задаю вопросы о проекте...
   Создаю .planning/PROJECT.md, REQUIREMENTS.md, ROADMAP.md...

Вы: gsd:discuss-phase 1 --analyze
Qwen: Анализирую компромиссы, создаю tradeoff таблицу...

Вы: gsd:plan-phase 1
Qwen: Создаю детальный план фазы 1...

Вы: gsd:ui-phase 1
Qwen: Генерирую UI-SPEC.md с компонентами и состояниями...

Вы: gsd:execute-phase 1
Qwen: Выполняю планы параллельными волнами...

Вы: gsd:ui-review 1
Qwen: Провожу визуальный аудит по 6 направлениям...

Вы: gsd:verify-work
Qwen: Проверяю работу, готовлю UAT...

Вы: gsd:ship
Qwen: Создаю релиз/PR...

Вы: gsd:next
Qwen: Определяю следующее: gsd:plan-phase 2
```

```
                    │  gsd:new-project│

                    └────────┬────────┘
                             ↓
              ┌──────────────┴──────────────┐
              ↓                              ↓
     ┌─────────────────┐           ┌─────────────────┐
     │gsd:discuss-phase│           │  gsd:ui-phase   │
     │    [--analyze]  │           │                 │
     └────────┬────────┘           └────────┬────────┘
              ↓                              ↓
              └──────────────┬──────────────┘
                             ↓
                    ┌─────────────────┐
                    │ gsd:plan-phase  │
                    └────────┬────────┘
                             ↓
                    ┌─────────────────┐
                    │gsd:execute-phase│
                    └────────┬────────┘
                             ↓
                    ┌─────────────────┐
                    │  gsd:ui-review  │
                    └────────┬────────┘
                             ↓
                    ┌─────────────────┐
                    │ gsd:verify-work │
                    └────────┬────────┘
                             ↓
                    ┌─────────────────┐
                    │    gsd:ship     │

                             ↓
                    ┌─────────────────┐
                    │     gsd:next    │──→ Повторить для следующей фазы
                    └─────────────────┘
```
