# GSD для Qwen Code — Полная документация

Адаптированная версия **Get Shit Done v1.29.0** для Qwen Code.

---

## 📌 Описание

**GSD (Get Shit Done)** — легковесная система мета-промптинга, контекстной инженерии и spec-driven разработки.

**Решает проблему:** context rot — деградация качества кода по мере заполнения контекстного окна.

**Для Qwen Code:** Ручная адаптация — пишите команды текстом в чат.

---

## 🚀 Быстрый старт

```bash
# 1. Инициализация проекта
gsd:new-project

# 2. Для существующего проекта
gsd:map-codebase

# 3. Проверка статуса
gsd:progress

# 4. Следующий шаг
gsd:next
```

## 📋 Все команды

### 🔷 CORE WORKFLOW (Основные)

| Команда | Описание | Флаги |
|---------|----------|-------|
| `gsd:new-project` | Инициализация: вопросы → research → требования → roadmap | — |
| `gsd:map-codebase` | Анализ существующей кодовой базы | `[area]` |
| `gsd:discuss-phase [N]` | Обсуждение решений реализации | `--auto`, `--analyze` |
| `gsd:plan-phase [N]` | Исследование + план + верификация | — |
| `gsd:execute-phase [N]` | Выполнение планов параллельными волнами | — |
| `gsd:verify-work [N]` | UAT тестирование | — |
| `gsd:ship [N]` | Создание релиза/PR | — |
| `gsd:next` | Авто-переход к следующему шагу | — |

### 🔷 UI DESIGN

| Команда | Описание | Флаги |
|---------|----------|-------|
| `gsd:ui-phase [N]` | Генерация UI дизайн-контракта (UI-SPEC.md) | `--detailed` |
| `gsd:ui-review [N]` | Ретроспективный визуальный аудит (6 столпов) | `--strict`, `--report` |

### 🔷 NAVIGATION (Навигация)

| Команда | Описание |
|---------|----------|
| `gsd:progress` | Где я? Что дальше? |
| `gsd:next` | Авто-определение состояния и запуск следующего шага |
| `gsd:manager` | Интерактивный центр управления фазами |
| `gsd:manager --interactive` | Полноценный интерактивный режим |
| `gsd:manager --overview` | Dashboard проекта без меню |
| `gsd:help` | Показать все команды и руководство |
| `gsd:update` | Обновить GSD |
| `gsd:settings` | Конфигурация профиля модели и агентов |
| `gsd:set-profile <profile>` | Переключение профиля (quality/balanced/budget/inherit) |
| `gsd:stats` | Статистика проекта — фазы, планы, требования, git метрики |
| `gsd:health` | Валидация .planning/ директории |
| `gsd:health --repair` | Автоматическое исправление проблем |

### 🔷 QUICK TASKS (Быстрые задачи)

| Команда | Описание | Флаги |
|---------|----------|-------|
| `gsd:quick [задача]` | Быстрое выполнение ad-hoc задач | — |
| `gsd:quick [задача] --discuss` | С обсуждением перед планированием | `--discuss` |
| `gsd:quick [задача] --research` | С исследованием подходов | `--research` |
| `gsd:quick [задача] --full` | С plan-checking + верификацией | `--full` |
| `gsd:do [задача]` | Выполнить конкретную задачу | — |
| `gsd:note [заметка]` | Захват идей без трения | — |

### 🔷 PHASE MANAGEMENT (Управление фазами)

| Команда | Описание |
|---------|----------|
| `gsd:add-phase [N] [name]` | Добавить фазу в roadmap |
| `gsd:insert-phase [N] [name]` | Вставить срочную фазу (decimal, e.g. 2.1) |
| `gsd:remove-phase [N]` | Удалить будущую фазу |
| `gsd:pause-work` | Создание handoff при остановке |
| `gsd:resume-work` | Восстановление из последней сессии |

### 🔷 RESEARCH & AUDIT (Исследования и аудит)

| Команда | Описание |
|---------|----------|
| `gsd:research-phase [N]` | Исследование домена для фазы N |
| `gsd:audit-milestone [N]` | Аудит завершённого milestone |
| `gsd:audit-uat [N]` | Детальный UAT аудит |
| `gsd:forensics [описание]` | Расследование неудачных запусков |

### 🔷 CODE QUALITY (Качество кода)

| Команда | Описание |
|---------|----------|
| `gsd:debug [описание]` | Систематическая отладка |
| `gsd:review [N]` | Code review для фазы N |
| `gsd:validate-phase [N]` | Валидация выполненной фазы |

### 🔷 BACKLOG & THREADS (Бэклог и потоки)

| Команда | Описание |
|---------|----------|
| `gsd:plant-seed <идея>` | Захват идей с условиями активации |
| `gsd:add-backlog <описание>` | Добавить идею в бэклог |
| `gsd:review-backlog` | Ревизия и продвижение элементов бэклога |
| `gsd:thread [name]` | Постоянные контекстные потоки |

### 🔷 WORKSTREAMS (Параллельные потоки)

| Команда | Описание |
|---------|----------|
| `gsd:workstreams list` | Показать все workstreams |
| `gsd:workstreams create <name>` | Создать именованный workstream |
| `gsd:workstreams switch <name>` | Переключить активный workstream |
| `gsd:workstreams complete <name>` | Завершить и объединить workstream |

### 🔷 COMMUNITY (Сообщество)

| Команда | Описание |
|---------|----------|
| `gsd:join-discord` | Присоединиться к Discord сообществу |

## 📁 Структура проекта

```
.planning/
├── PROJECT.md              # Видение проекта (всегда загружается)
├── REQUIREMENTS.md         # Требования с трассировкой по фазам
├── ROADMAP.md              # Дорожная карта, статус выполнения
├── STATE.md                # Решения, блокеры, позиция — память между сессиями
├── config.json             # Настройки проекта
├── codebase/               # Анализ кодовой базы
│   ├── STACK.md
│   ├── ARCHITECTURE.md
│   ├── STRUCTURE.md
│   ├── CONVENTIONS.md
│   ├── TESTING.md
│   └── CONCERNS.md
├── ui-specs/               # UI спецификации
│   └── phase-N-UI-SPEC.md
├── audits/                 # Аудиты
│   └── ui-review-phase-N.md
├── quick/                  # Быстрые задачи
│   └── QUICK-NN/
│       ├── CONTEXT.md
│       ├── RESEARCH.md
│       ├── PLAN.md
│       └── SUMMARY.md
├── todos/                  # Идеи и задачи на потом
├── threads/                # Постоянные контекстные потоки
├── seeds/                  # Идеи с условиями активации
└── {phase}-*.md            # Документы фаз (CONTEXT, RESEARCH, PLAN, SUMMARY, VERIFICATION, UAT)
```

## 🔄 Основной Workflow

```
┌─────────────────────────────────────────────────────────────────┐
│  1. gsd:new-project (для нового)                                │
│     или gsd:map-codebase (для существующего)                    │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  2. gsd:discuss-phase [N] [--analyze]                           │
│     Обсуждение решений → CONTEXT.md                             │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  3. gsd:plan-phase [N]                                          │
│     Исследование → Планы → Верификация                          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  4. gsd:execute-phase [N]                                       │
│     Выполнение волнами → Атомарные коммиты                      │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  5. gsd:ui-phase [N]                                            │
│     UI спецификация (UI-SPEC.md)                                │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  6. gsd:ui-review [N]                                           │
│     Визуальный аудит по 6 направлениям                          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  7. gsd:verify-work [N]                                         │
│     UAT тестирование → Диагностика                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  8. gsd:ship [N]                                                │
│     Публикация работы                                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  9. gsd:next                                                    │
│     Переход к следующей фазе                                    │
└─────────────────────────────────────────────────────────────────┘
```

## ⚙️ Конфигурация

### Model Profiles

| Профиль | Planning | Execution | Verification | Стоимость |
|---------|----------|-----------|--------------|-----------|
| `quality` | Opus | Opus | Sonnet | $$$ |
| `balanced` (default) | Opus | Sonnet | Sonnet | $$ |
| `budget` | Sonnet | Sonnet | Haiku | $ |
| `inherit` | Inherit | Inherit | Inherit | — |

```bash
gsd:set-profile quality
gsd:set-profile balanced
gsd:set-profile budget
gsd:set-profile inherit
```

### Настройки workflow

```json
{
  "workflow": {
    "research": true,           // Исследование перед планированием
    "plan_check": true,         // Верификация планов
    "verifier": true,           // Подтверждение доставки
    "auto_advance": false,      // Авто-цепочка шагов
    "discuss_mode": "discuss"   // "discuss" или "assumptions"
  },
  "execution": {
    "parallelization": {
      "enabled": true           // Параллельное выполнение
    }
  }
}
```

## 💡 Уникальные функции GSD

### 1️⃣ Контекстная инженерия
- Разделение на под-агентов со свежим контекстом
- Основное окно остаётся на 30-40%
- Работа в свежих контекстах

### 2️⃣ Волновое выполнение
```
WAVE 1 (parallel)     WAVE 2 (parallel)     WAVE 3
┌─────────┐ ┌───────┐  ┌─────────┐ ┌───────┐  ┌─────────┐
│ Plan 01 │ │ Plan 02│→ │ Plan 03 │ │ Plan 04│→ │ Plan 05 │
└─────────┘ └───────┘  └─────────┘ └───────┘  └─────────┘
```

### 3️⃣ Атомарные Git коммиты
```
abc123f docs(01-02): complete user registration plan
def456g feat(01-02): add email confirmation flow
hij789k feat(01-02): implement password hashing
```

### 4️⃣ Discuss Mode с trade-off анализом
```bash
gsd:discuss-phase 1 --analyze  # Глубокий анализ компромиссов
```

### 5️⃣ Quick Mode для ad-hoc задач
```bash
gsd:quick --discuss --research --full  # Полный режим
```

### 6️⃣ Seed System (Идеи с триггерами)
```bash
gsd:plant-seed "Добавить кэширование когда >1000 пользователей"
```

### 7️⃣ Workstreams для параллельной работы
```bash
gsd:workstreams create feature-a
gsd:workstreams switch feature-a
```

## 🛠️ Troubleshooting

| Проблема | Решение |
|----------|---------|
| Команда не найдена | Проверьте `.qwen/gsd/workflows/` |
| .planning/ повреждён | `gsd:health --repair` |
| STATE.md не соответствует | `gsd:health --repair` |
| Нужно начать заново | Удалить `.planning/`, запустить `gsd:new-project` |
